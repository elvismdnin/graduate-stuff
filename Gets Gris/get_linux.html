GET-LINUX

1. Explique com suas palavras o que faz o comando utilizado para instalar o Arch Linux “arch-chroot”.
O comando [arch-chroot] mantém as configurações e abre um shell que simula um sistema começado na pasta escolhida, ele abre um shell que acredita que o '/' do sistema de arquivos está na pasta do comando aberto.
2. Como funciona o comando do terminal sudo?
Basicamente ele cria uma instancia do bash e seta uid do usuário para executar um código que é recebido de entrada. Fazendo isso, ele pode fazer o programa acreditar que ele tem os direitos de root, sem sair ou trocar o usuário.
3. Em um servidor web precisou-se dar acesso para que um usuário execute seus arquivos “.php” em sua pasta public_html, qual a permissão mais restritiva em que ele possa fazer isso? E da pasta onde ele está? (OBS: servidor web é um outro usuário no Linux).
Admitindo que o usuário servidor precise de todo acesso a permissão mais restritiva seria a 701, que liberaria o usuário de fora do grupo execute o .php.
Já da pasta, na mesma ideia, o usuário de fora teria que ter direito a executar para abrir a pasta, e direito de ler, para ver o .php que está dentro dela, logo 705, o padrão das pastas.
Admite-se que o dono das pastas é o usuário servidor local.
4. Detalhe 3 Shell’s e suas respectivas diferenças.
  1. BASH: Bourne-Again Shell, o shell padrão do GNU, ele é simples e poderoso. Você sempre pode contar com ele.
  2. ZSH: Baseado em BASH com inclusão de algumas features como modificar o texto vim-like e autocomplete muito esperto.
  3. CSH: A intenção era fazer um shell onde os scripts parecessem mais com C, o que os torna mais legíveis.
5. Qual a diferenças entre um serviço no Linux e um processo?
Um serviço é um pacote completo que resolve algum problema. Um webserver é um serviço, ele serve páginas web.
Um processo, por outro lado, é uma abstração em cima de um código, feita pelo OS, que tem variáveis de ambiente. É criado com a execução de um código.
Um serviço pode ter vários processos.
6. Por que é errado chamar o Linux de Sistema Operacional?
Porque o Linux é o kernel, a parte que resolve uma parte dos problemas que um OS se propõe a resolver. OS são as distribuições do linux, que trazem vários modos de interagir com o computador através do kernel.
7. Para que servem as entradas na árvore de diretórios do Linux:
  1. /dev/stdout
    O arquivo que representa a saída padrão do terminal
  2. /tmp/.X11-unix
    Pasta de sockets, que representam as relações dos programas no sistema com o server X, server de aplicações gráficas.
  3. /proc/cpuinfo
    O arquivo com as descrições de cada um dos cores do processador.
  4. /etc/shadow
    O arquivo que guarda criptografadas todas as senhas dos usuários.
  5. /var/spool
    Pasta onde são guardadas as coisas que estão em fila para serem entregadas, como emails ou documentos para impressão
  6. /etc/fstab
    Arquivo com definições dos sistemas de arquivos montados ao iniciar
8. Qual a diferença entre o /tmp/.X11-unix para as outras entradas?
A pasta do X11 tem vários sockets, que são arquivos de interface, eles não são arquivos, como os outros, eles são descritores de regras de negócio.
9. O que são links no Linux? E qual(is) a(s) diferença(s) entre um hard link e um soft link?
Links são modos de relacionar arquivos no sistema de arquivos ligando-os. O hard link aponta para o arquivo em si, no hd. O soft link aponta para o descritor do arquivo. Mudanças no nome do arquivo quebram o soft link, e não o hard.
10. Por que ao se ligar um dispositivo periférico a um sistema computacional com Linux ele deve ser montado para poder ser utilizado?
Para que o sistema saiba como lidar com o arquivo, já que o dispositivo periférico vem como um arquivo. Ele poderia ser qualquer coisa, ao montar dizemos o que ele é e como usaremos.
11. Qual a função da variável de ambiente PATH?
A variável PATH descreve os caminhos onde procurar os programas que são acessíveis diretamente do shell. Ela pode ser usada para definir um local padrão novo, de programas, ao se dar export.
12. Quais as vantagens de se colocar o diretório /home em uma outra partição separada do “/” ?
Se eu tiver o /home separado do / eu posso fazer um sistema de arquivos em rede e colocar a pasta de configurações de cada usuário acessível a várias máquinas diferentes.
13. O que são os sistemas de init? Quais as diferenças entre o sysvinit e o systemd?
init é um sistema que gerencia os serviços através de scripts feitos em bash, e executados com alguns comandos específicos. O systemd tem a intenção de melhorar esse processo usando, além de outras estratégias, um modo próprio de fazer esses scripts e mostrando os serviços em interfaces mais trabalhadas.
14. Pra que servem as flags especiais de permissão do Linux setuid, setgid e sticky bit?
  1. SetUID: é um sinalizador de direito que define se você pode executar determinado arquivo com permissões de outro usuário.
  2. SetGID: é quase a mesma coisa do de cima, só que define se voce pode executar como outro grupo.
  3. Sticky Bit: é um indicador que diz o que for feito a partir dali carregará suas permissões, muito útil para diretórios, onde você deixa o usuário criar, mas, mexer apenas nas suas coisas.
